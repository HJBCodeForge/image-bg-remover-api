API Documentation

Overview

The Background Remover API provides AI-powered background removal capabilities for images. This RESTful API allows you to remove backgrounds from images with high precision and speed, designed for integration into web applications, mobile apps, and automated workflows.

  • Base URL: https://web-production-faaf.up.railway.app
  • API Version: 1.0.0
  • Content Type: multipart/form-data
  • Response Format: JSON/Binary
  • Supported Formats: JPEG, PNG, WebP, BMP, TIFF
  • Max File Size: 5MB
  • Processing Time: 1-5 seconds
  • Rate Limiting: No limits (usage tracked)

Authentication

The API uses API key authentication. You must include your API key in requests to access protected endpoints.

Authentication Methods

Method 1: Form Data (Recommended)

// Include in form data
formData.append('api_key', 'bgr_your_api_key_here');

Method 2: Authorization Header

// Include in request headers
Authorization: Bearer bgr_your_api_key_here
Note: API keys are generated using the /api-keys endpoint and start with the prefix bgr_. Keep your API key secure and never expose it in client-side code.

Quick Start Guide

1. Generate API Key

Create a new API key using the POST /api-keys endpoint with a descriptive label to identify your application or project.

2. Send Image

Send your image file to the POST /remove-background endpoint with your API key and optional parameters.

3. Get Result

Receive the processed image with transparent background in PNG format, ready for use in your application.

Remove Background

POST /remove-background

Remove background from an image using AI processing.

Request Parameters:

  • file (file, required) - Image file to process
  • api_key (string, required) - Your API key
  • alpha_matting (boolean, optional) - Enable alpha matting for better edges (default: false)
  • alpha_matting_foreground_threshold (integer, optional) - Foreground threshold 0-255 (default: 270)
  • alpha_matting_background_threshold (integer, optional) - Background threshold 0-255 (default: 10)
  • alpha_matting_erode_structure_size (integer, optional) - Erosion size 1-20 (default: 10)
  • alpha_matting_base_size (integer, optional) - Base size 500-2000 (default: 1000)

Response: Binary PNG image data with transparent background

Code Examples

JavaScript/Node.js

// Generate API Key
const generateApiKey = async (keyName) => {
  const formData = new FormData();
  formData.append('key_name', keyName);
  
  const response = await fetch('https://web-production-faaf.up.railway.app/api-keys', {
    method: 'POST',
    body: formData
  });
  
  return await response.json();
};

// Remove Background
const removeBackground = async (imageFile, apiKey) => {
  const formData = new FormData();
  formData.append('file', imageFile);
  formData.append('api_key', apiKey);
  formData.append('alpha_matting', 'true');
  
  const response = await fetch('https://web-production-faaf.up.railway.app/remove-background', {
    method: 'POST',
    body: formData
  });
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  return await response.blob(); // Returns processed image
};

Python

import requests

# Generate API Key
def generate_api_key(key_name):
    response = requests.post(
        'https://web-production-faaf.up.railway.app/api-keys',
        data={'key_name': key_name}
    )
    return response.json()

# Remove Background
def remove_background(image_path, api_key):
    with open(image_path, 'rb') as image_file:
        files = {'file': image_file}
        data = {
            'api_key': api_key,
            'alpha_matting': 'true'
        }
        
        response = requests.post(
            'https://web-production-faaf.up.railway.app/remove-background',
            files=files,
            data=data
        )
        
        if response.ok:
            return response.content  # Binary image data
        else:
            raise Exception(f"Error: {response.status_code}")

Error Handling

The API returns standard HTTP status codes and JSON error responses:

Common HTTP Status Codes

  • 200 OK - Request successful
  • 400 Bad Request - Invalid request parameters
  • 401 Unauthorized - Invalid or missing API key
  • 413 Payload Too Large - File exceeds 5MB limit
  • 500 Internal Server Error - Server processing error

Error Response Format

{
  "detail": "Invalid API key"
}

Error Handling Example

try {
  const result = await removeBackground(imageFile, apiKey);
  // Handle success
} catch (error) {
  if (error.response) {
    console.error('API Error:', error.response.data.detail);
  } else {
    console.error('Network Error:', error.message);
  }
}

Alpha Matting Options

Alpha matting provides superior edge quality by analyzing pixel transparency. Enable it for professional results:

  • Foreground Threshold (0-255): Controls foreground pixel detection. Higher values = more selective
  • Background Threshold (0-255): Controls background pixel detection. Lower values = more selective
  • Erosion Size (1-20): Controls edge smoothing. Higher values = smoother edges
  • Base Size (500-2000): Processing resolution. Higher values = better quality, slower processing

Best Practices

Security

  • Never expose API keys in client-side code
  • Use environment variables for API keys
  • Implement rate limiting on your end

Performance

  • Optimize image size before uploading
  • Use alpha matting only when needed
  • Implement caching for processed images

Error Handling

  • Always handle HTTP errors gracefully
  • Implement retry logic for network issues
  • Validate file types before uploading
