API Documentation

Overview

The Background Remover API provides AI-powered background removal capabilities for images. This RESTful API allows you to remove backgrounds from images with high precision and speed.

Base URL: https://web-production-faaf.up.railway.app

Authentication: API Key required for image processing endpoints

Supported Formats: JPEG, PNG, WebP, BMP, TIFF

Max File Size: 5MB

Authentication

Most endpoints require an API key for authentication. Include your API key in the request headers:

Authorization: Bearer YOUR_API_KEY
Note: API keys are generated using the /api-keys endpoint and start with the prefix bgr_

API Key Management

Method Endpoint Description
POST /api-keys Create a new API key
GET /api-keys List all API keys
DELETE /api-keys/{id} Delete an API key

Create API Key Request:

POST /api-keys
Content-Type: application/json

{
  "name": "My Application"
}

Background Removal

Method Endpoint Description
POST /remove-background Remove background from image

Request Parameters:

Parameter Type Required Description
file File Yes Image file to process
return_json Boolean No Return JSON response instead of binary

Request Example:

POST /remove-background?return_json=true
Authorization: Bearer bgr_1234567890abcdef
Content-Type: multipart/form-data

file: [binary image data]

Response Example (JSON):

{
  "success": true,
  "message": "Background removed successfully",
  "processed_image_url": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...",
  "processing_time": 2.34
}

Error Handling

The API uses standard HTTP status codes and returns detailed error information:

Status Code Meaning Description
200 OK Request successful
400 Bad Request Invalid request parameters
401 Unauthorized Missing or invalid API key
413 Payload Too Large File size exceeds 5MB limit
500 Internal Server Error Server processing error

Error Response Format:

{
  "success": false,
  "error": "Invalid API key",
  "details": "The provided API key is not valid or has been revoked"
}

Rate Limits & Best Practices

  • No rate limits currently enforced
  • File size limit: 5MB maximum
  • Supported formats: JPEG, PNG, WebP, BMP, TIFF
  • Processing time: Typically 1-5 seconds depending on image size

Best Practices:

  • Always validate file size before uploading
  • Handle timeout errors gracefully (set 120s timeout)
  • Store API keys securely, never expose in client-side code
  • Use the return_json=true parameter for better error handling
  • Monitor API key usage through the /api-keys endpoint
